Spiral Model - Project Development

The Spiral Model integrates iterative development with structured phases from the Waterfall model, emphasizing continuous risk analysis and refinement. The following outlines how our Disk Scheduling Algorithm Visualizer was developed using this methodology:

1. Planning & Requirement Gathering

During the initial phase, we defined objectives and gathered comprehensive requirements, focusing on:

  • Accepting disk I/O requests and an initial head position
  • Providing a selection of disk scheduling algorithms (FCFS, SSTF, SCAN, LOOK, etc.)
  • Delivering output metrics such as total seek time and head movement order

2. Risk Analysis & Prototype Design

We assessed potential technical risks and created low-fidelity prototypes to validate core concepts:

  • Explored animation strategies for visualizing head movement
  • Identified challenges related to dynamic and real-time visualization
  • Developed basic UI mockups for stakeholder review

3. Development & First Iteration

The first iteration focused on implementing core functionality based on gathered requirements:

  • Designed a responsive UI using HTML and CSS
  • Developed initial JavaScript modules for FCFS and SSTF algorithms
  • Introduced a basic canvas-based visualization for disk head movement

4. Evaluation & Refinement

Following each iteration, we incorporated feedback and enhanced the system for better performance:

  • Added support for additional algorithms like SCAN and LOOK
  • Improved animation for smoother transitions
  • Strengthened input validation and error handling

5. Further Iterations & Final Deployment

In the final phase, we focused on optimization and deployment readiness:

  • Conducted comprehensive responsiveness and cross-device testing
  • Resolved bugs identified during extensive test cases
  • Published the project on GitHub and prepared for live deployment

6. Maintenance & Future Enhancements

Post-deployment, the Spiral Model supports ongoing maintenance and continuous improvement:

  • Gathered user feedback to plan upcoming features
  • Outlined potential algorithm additions and UI improvements
  • Maintained modular code architecture for future scalability
